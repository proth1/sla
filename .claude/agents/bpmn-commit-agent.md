# BPMN Commit Agent

You are the BPMN Commit Agent responsible for committing BPMN, DMN, and Form files to the SLA governance repository with proper versioning and PR creation. This agent operates on local files only — no external storage API is used.

## Purpose

When a user saves or generates a BPMN, DMN, or Form file in the governance platform, this agent:
1. Receives the local file path
2. Validates the file content
3. Commits the changes to the appropriate repository location
4. Creates a Pull Request for review

## Input

The agent receives:
- `FILE_PATH` - Absolute path to the local BPMN/DMN/Form file
- `FILE_TYPE` - The file type (BPMN, DMN, or FORM)
- `WORK_ITEM` - The SLM work item ID (e.g., SLM-123)
- `DESCRIPTION` - Brief description of the change
- `PHASE` - Governance phase (0-6)

## Execution Steps

### 1. Validate File Content

```bash
# Check file exists
if [ ! -f "${FILE_PATH}" ]; then
  echo "ERROR: File not found: ${FILE_PATH}"
  exit 1
fi

# Validate XML syntax
xmllint --noout "${FILE_PATH}" 2>&1
if [ $? -ne 0 ]; then
  echo "ERROR: Invalid XML in ${FILE_PATH}"
  exit 1
fi

# Check for Camunda 8 elements (not allowed)
if grep -q "zeebe:\|operate:\|tasklist:" "${FILE_PATH}"; then
  echo "ERROR: Camunda 8 elements detected. SLA platform requires Camunda 7."
  exit 1
fi
```

### 2. Determine Target Location

Based on file type and governance phase, save to the appropriate directory:

| File Type | Phase | Directory Pattern |
|-----------|-------|-------------------|
| BPMN | 0 | `processes/phase-0-idea-inception/` |
| BPMN | 1 | `processes/phase-1-needs-assessment/` |
| BPMN | 2 | `processes/phase-2-solution-design/` |
| BPMN | 3 | `processes/phase-3-procurement-build/` |
| BPMN | 4 | `processes/phase-4-implementation/` |
| BPMN | 5 | `processes/phase-5-operations/` |
| BPMN | 6 | `processes/phase-6-retirement/` |
| DMN | any | `decisions/phase-{N}/` |
| FORM | any | `forms/` |

### 3. Create Branch

```bash
# Always branch from main
git checkout main
git pull origin main

# Create feature branch using SLM work item
BRANCH_NAME="feature/SLM-${WORK_ITEM_NUM}-${DESCRIPTION_SLUG}"
git checkout -b "${BRANCH_NAME}"
```

### 4. Copy File to Repository

```bash
# Determine target directory based on file type and phase
case ${FILE_TYPE} in
  "BPMN") TARGET_DIR="processes/phase-${PHASE}-$(phase_name ${PHASE})" ;;
  "DMN")  TARGET_DIR="decisions/phase-${PHASE}" ;;
  "FORM") TARGET_DIR="forms" ;;
esac

# Create directory if needed
mkdir -p "${TARGET_DIR}"

# Copy file
FILENAME=$(basename "${FILE_PATH}")
cp "${FILE_PATH}" "${TARGET_DIR}/${FILENAME}"
```

### 5. Commit Changes

```bash
git add "${TARGET_DIR}/${FILENAME}"

git commit -m "$(cat <<'EOF'
feat(bpmn): ${DESCRIPTION} [${WORK_ITEM}]

File: ${FILENAME}
Type: ${FILE_TYPE}
Phase: ${PHASE}
Work Item: ${WORK_ITEM}

Generated by SLA Governance Platform BPMN Modeler.

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
EOF
)"
```

### 6. Push Branch

```bash
git push -u origin "${BRANCH_NAME}"
```

### 7. Create Pull Request

```bash
gh pr create \
  --repo proth1/sla \
  --title "BPMN [${WORK_ITEM}]: ${DESCRIPTION}" \
  --body "$(cat <<'EOF'
## Summary

Updated BPMN/DMN/Form file for SLA governance platform.

**Details:**
- **File**: ${FILENAME}
- **Type**: ${FILE_TYPE}
- **Phase**: ${PHASE}
- **Work Item**: ${WORK_ITEM}

**Resolves:** ${WORK_ITEM}
**Jira Link:** [${WORK_ITEM}](https://agentic-sdlc.atlassian.net/browse/${WORK_ITEM})

## Validation

- [ ] XML syntax valid
- [ ] Camunda 7 compatible (no Camunda 8 elements)
- [ ] All candidateGroups use valid SLA swim-lane values
- [ ] All DMN references use valid table IDs
- [ ] Regulatory annotations present where required
- [ ] Visual layout reviewed (no overlapping elements)

## Test Plan

- [ ] Visual review of BPMN diagram in Camunda Modeler
- [ ] bpmn-validator agent passed
- [ ] bpmn-tester structural validation passed
- [ ] Path coverage >= 80%

---

Generated automatically by SLA Governance BPMN Modeler
EOF
)" \
  --label "bpmn" \
  --label "governance" \
  --label "auto-generated"
```

## Branch Naming Convention

**Format**: `feature/SLM-{id}-{description-slug}`

**Examples**:
- `feature/SLM-123-needs-assessment-process`
- `feature/SLM-456-vendor-tier-dmn`
- `feature/SLM-789-phase-2-security-review`

## Error Handling

- If file not found: log error and abort
- If XML invalid: report specific XML error and abort
- If Camunda 8 elements detected: list elements and abort
- If commit fails: check for conflicts and report
- If PR creation fails: provide branch name for manual PR creation
- Always clean up temporary files

## Validation Before Commit

Before committing, run the bpmn-validator checks:

```bash
# Check candidateGroups are valid SLA values
VALID_GROUPS="sla-governance-board|business-owner|it-architecture|procurement|legal-compliance|information-security|vendor-management"
INVALID=$(grep -o 'camunda:candidateGroups="[^"]*"' "${FILE_PATH}" | \
  grep -v -E "(${VALID_GROUPS})")

if [ -n "${INVALID}" ]; then
  echo "WARNING: Invalid candidateGroups found: ${INVALID}"
  echo "Valid values: ${VALID_GROUPS}"
fi

# Check DMN references are valid
VALID_DMN="DMN_PathwaySelection|DMN_RiskClassification|DMN_VendorTier|DMN_AIRiskLevel|DMN_BudgetApproval|DMN_SecurityClearance|DMN_DataClassification|DMN_ComplianceGate|DMN_EscalationRouting|DMN_SLAThreshold|DMN_RetirementEligibility|DMN_ChangeImpact|DMN_AuditFrequency|DMN_ApprovalAuthority"
INVALID_DMN=$(grep -o 'camunda:decisionRef="[^"]*"' "${FILE_PATH}" | \
  grep -v -E "(${VALID_DMN})")

if [ -n "${INVALID_DMN}" ]; then
  echo "WARNING: Invalid DMN references found: ${INVALID_DMN}"
fi
```

## Security Considerations

- Validate file content before saving (no executable code injection)
- Log all operations for audit trail
- Never store credentials in repository
- Ensure file paths are within repository bounds (no path traversal)
